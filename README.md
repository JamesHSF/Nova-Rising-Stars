# 摘星計畫 ✦ 團隊打卡系統

三級權限的團隊打卡 web app：**一般成員**（每日打卡／每週目標／每月復盤）、**組長**（建組、看達成率、提醒組員，看不到內容）、**管理者**（分組、設定當組管理員，只有當組管理員能檢視與匯出該組資料）。

## 檔案

| 檔案 | 用途 |
|---|---|
| `index.html` | 整個 app（單一檔案，零相依） |
| `supabase-setup.sql` | 後端資料表與權限規則，一鍵建立 |

## 快速體驗（示範模式）

直接雙擊開啟 `index.html` 即可。未設定 Supabase 時自動進入示範模式：
- 資料只存在這台瀏覽器（localStorage）
- 登入頁有「一鍵載入示範帳號」：成員／組長／管理者各一，密碼都是 `123456`
- 第一位自行註冊的人自動成為管理者

## 正式上線（三步驟）

1. **建立 Supabase 專案**（免費）：到 [supabase.com](https://supabase.com) 註冊 → New Project。
2. **建立資料庫**：Dashboard → SQL Editor → 貼上 `supabase-setup.sql` 全部內容 → Run。
3. **填入金鑰**：Dashboard → Project Settings → API，複製兩個值填入 `index.html` 最上方：
   ```js
   const CONFIG = {
     SUPABASE_URL: 'https://xxxx.supabase.co',
     SUPABASE_ANON_KEY: 'eyJ...',
     GOOGLE_CALENDAR_EMBED: '',  // 選填，見下方
   };
   ```

之後把 `index.html` 放上任何靜態空間即可（Netlify Drop 拖曳上傳、GitHub Pages、Vercel 都行）。第一位註冊的人自動成為管理者。

> 註：Supabase 預設註冊需 Email 驗證。若想省略，到 Authentication → Providers → Email 關閉「Confirm email」。

## 團隊行事曆（每月復盤頁，使用 Google 日曆）

Google 日曆 → 設定 → 選擇日曆 → 整合日曆 → 複製「嵌入代碼」iframe 的 `src` 網址，貼到 `CONFIG.GOOGLE_CALENDAR_EMBED`。日曆需設為公開或使用共用連結。

## 權限設計（由資料庫 RLS 強制，不是只有前端擋）

| 角色 | 能做什麼 |
|---|---|
| 一般成員 | 讀寫自己的每日／每週盤點／每月紀錄；收提醒；看自己的摘星軌跡 |
| 守護者（組長） | ＋建立組別；看組員「已填／未填」、打卡率、每週黃星星／紫星軌跡；可展開組員**每日／每週的完整填寫內容**（RLS `is_team_leader_of_user()` 放行，不含每月）；寄提醒 |
| 業務組別管理員 | 被指派後可 SELECT 該組成員完整紀錄與每日／每週／每月狀況（檢視＋匯出 CSV）。一位成員可同時屬於多個業務組別（`group_members` 多對多），各組管理員都看得到他 |
| 最高管理員（第一位註冊者） | ＋建立業務組別並命名、指派各組的業務組別管理員、調整所有人的角色與分組；可檢視所有組別 |

> 角色分兩層：`profiles.role`（member／leader／admin）與 `profiles.is_super`（最高管理員旗標，第一位註冊者自動取得）。業務組別管理員＝`role='admin'` 且被設為某組 `admin_id` 的人。

## 每週盤點週期與星星

每週盤點以「**週六～週五**」為一週，且盤點的是**已結束的上一週**（例：7/11 填的是 7/4–7/10）。

四種星星：
- **🌟 黃星**（個人・每日）：合格打卡一天（業務≥低標＋專注正向滋養≥1＋心情日記）得 1 顆。
- **✦ 紫星**（個人・每週）：整週集滿 7 顆黃星（上線首週 5 顆）且盤點 BV 高於上週、完成兩訪、列滿 TOP 10 名單。
- **🟠 橘星**（團隊・每日）：組長組別**全員**當日都得黃星 → 該日團隊得 1 顆。
- **🌈 彩虹星**（團隊・每週）：組長組別**全員**該週都得紫星 → 該週團隊得 1 顆。

橘星／彩虹星顯示在組長面板；Supabase 由 `team_bonus_stars()` RPC 計算。

## 最高管理員統計報表

管理面板底部（僅最高管理員）提供統計報表，可切換 今日／本週／本月／本季：
黃星分布（不同數量各有哪些人）、紫星分布、各組別四種星星排行、成員與組別的 BV／IBV／簽約店長／再推薦排行。資料由前端彙整（最高管理員可讀全部紀錄，RLS 以 `am_super()` 放行）。
